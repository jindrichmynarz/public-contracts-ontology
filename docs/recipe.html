Recept pro vyznačení veřejné zakázky v HTML stránce pomocí jazyka RDFa

Autoři:
  • Martin Nečaský (MFF UK)

Tento dokument názorně ukazuje, jakým způsobem lze zveřejňovat údaje o veřejné zakázce ve strojově čitelné podobě. Základem zveřejnění je klasická HTML stránka. Ta je doplněna o strojově čitelné anotace údajů vyjádřených pomocí prostředků rozšíření jazyka HTML označovaného zkratkou RDFa. Pro sjednocení podoby strojově čitelných údajů jsme navrhli slovník, který popisuje jednotlivé datové položky veřejné zakázky. Ty je potřeba dodržet. Nicméně, nemají vliv na samotnou vizuální podobu HTML stránky.

Kompletní příklad veřejné zakázky zveřejněné v podobě HTML stránky doplněné o strojově čitelné anotace vyjádřené v jazyce RDFa najdete ZDE.

Základní struktura dokumentu

Pokud používáte formát XHTML, upravte jeho záhlaví a kořenový element html tak, aby bylo řečeno, že dokument je XHTML dokument se strojově čitelnými anotacemi:

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML+RDFa 1.1//EN" "http://www.w3.org/MarkUp/DTD/xhtml-rdfa-2.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="cs" lang="cs" version="XHTML+RDFa 1.1">
   …
<html>

Pokud používáte formát HTML, situace je obdobná. V záhlaví uvedete, že se jedná o HTML dokument se strojově čitelnými anotacemi:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01+RDFa 1.1//EN"
   "http://www.w3.org/MarkUp/DTD/html401-rdfa11-1.dtd">
<html>
   …
</html>

Jmenné prostory

Kořenový element html doplňte o deklarace jmenných prostorů. Snažíme se o kompatibilitu se všemi verzemi jazyka RDFa (1.0 a 1.1). Pro kompatibilitu s verzí RDFa 1.1 je potřeba zavádět jmenné prostory pomocí atributu prefix

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="cs" lang="cs" version="XHTML+RDFa 1.1"
      prefix="rdf: http://www.w3.org/1999/02/22-rdf-syntax-ns#
              rdfs: http://www.w3.org/2000/01/rdf-schema#
              gr: http://purl.org/goodrelations/v1#
              dc: http://purl.org/dc/terms/
              pc: http://purl.org/opendata-cz/public-contracts#
              pccz: http://purl.org/opendata-cz/public-contracts/czech#
              payment: http://reference.data.gov.uk/def/payment#
              xsd: http://www.w3.org/2001/XMLSchema#">
   …
</html>

Pro kompatibilitu s verzí RDFa 1.0 je potřeba zavádět jmenné prostory pomocí XML jmenných prostorů

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="cs" lang="cs" version="XHTML+RDFa 1.1"
      xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
      xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#"
      xmlns:gr="http://purl.org/goodrelations/v1#"
      xmlns:dc="http://purl.org/dc/terms/"
      xmlns:pc="http://purl.org/opendata-cz/public-contracts#"
      xmlns:pccz="http://purl.org/opendata-cz/public-contracts/czech#"
      xmlns:xsd="http://www.w3.org/2001/XMLSchema#">
   …
</html>

Pro kompatibilitu s oběma verzemi použijte oba způsoby najednou.

Proč je použito tolik jmenných prostorů? Slovník maximálně využívá části jiných, již hotových a široce používaných slovníků. To zajistí interoperabilitu dat o veřejných zakázkách s jinými daty vytvořenými dle těchto slovníků.

Určení místa na webové stránce, kde je veřejná zakázka popsána

Informace o veřejné zakázce uzavřete ve vašem HTML kódu do jednoho elementu, např. div. Potom určete, že obsah tohoto elementu se týká veřejné zakázky pomocí RDFa atributu typeof s hodnotou pc:Contract. Jinými slovy tím říkáte, že obsah reprezentuje instanci třídy pc:Contract.

<div typeof="pc:Contract">
  …
</div>

Výše uvedeným dále říkáte, že samotná veřejná zakázka je anonymní, tj. nemá žádný identifikátor. V RDFa je možné instance nepovinně identifikovat pomocí URI. Identifikační URI vám např. umožní se na veřejnou zakázku odkazovat z jiného místa na webu, kde máte umístěny nějaké další údaje o zakázce. Toto URI může být např. fyzickým URL, na kterém je umístěna HTML stránka s veřejnou zakázkou. Je důležité ale zajistit, že zvolené identifikační URI se pro danou veřejnou zakázku již nikdy nezmění.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
</div>

Výše uvedeným říkáte, že veřejná zakázka není anonymní. Je identifikována pomocí URL http://www.mesto.cz/VZ/12345678.

Údaje při zveřejnění veřejné zakázky

Pro každou veřejnou zakázku uvádějte její název. Pro názvy je v našem slovníku určena RDF vlastnost dc:title. Název umístěte do samostatného HTML elementu, např. span. Pomocí jeho RDFa atributu property s hodnotou dc:title řeknete, že obsah elementu je názvem veřejné zakázky.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   Název: <span property="dc:title">Jezdecké potřeby městské policie</span>
</div>

Dále uvádějte datum zveřejnění a datum pro posílání nabídek. Pro ně jsou v našem slovníku určeny RDF vlastnosti dc:issued a pc:bidDeadline. Podobně jako v případě názvu zakázky, uveďte každý datum v samostatném HTML elementu, např. span, který opatřete RDFa atributem property s hodnotou dc:issued, resp. pc:bidDeadline. Je zde ale jeden podstatný rozdíl oproti názvu. Na webové stránce chcete návštěvníkům zobrazit datum ve formátu běžném v ČR, např. 1. 2. 2010 nebo 1. února 2010. To ale není strojově čitelné datum. Pro strojově čitelnou podobu datumů používáme datový typ xsd:date jazyka XML Schema, např. 2010-02-01 (RRRR-MM-DD). Strojově čitelnou hodnotu ale pravděpodobně nechcete ukazovat svým návštěvníkům. Proto ji dávejte jako hodnotu RDFa atributu content. Také přidejte informaci o datovém typu pomocí atributu datatype.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   …
   Datum zveřejnění: <span property="dc:issued" content="2010-01-01" datatype="xsd:date">1. 1. 2010</span>
   Datum pro podávání nabídek: <span property="pc:bidDeadline" content="2010-02-01" datatype="xsd:date">1. 2. 2010</span>
</div>

Dalším údajem je předmět veřejné zakázky ve formě CPV kódu. Pro vyhledávání CPV kódů v rámci svého softwarového řešení můžete použít jednoduchý nástroj http://opendata.cz/cpv/. Předmětů může mít veřejná zakázka více. V našem slovníku je pro ně určena RDF vlastnost pc:subject. Oproti předchozím údajům ale nejsou CPV kódy jednoduché (atomické) datové hodnoty, ale samostatné entity identifikované vlastním URL. V RDF datovém modelu jsou entity nazývány zdroje. Opět je uvádíme v samostatných HTML elementech. Tentokrát však nemají tyto elementy RDFa atribut property ale RDFa atribut rel s hodnotou pc:subject. Tím říkáme, že hodnotou vlastnosti není jednoduchá datová hodnota ale zdroj, jehož URL uvádíme jako hodnotu atributu resource. CPV kód má URL http://opendata.cz/vocabulary/cpv/2008#cpv-[kod], kde [kod] nahraďte samotným CPV kódem (např. 18911000-8).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   …
   Předmět: <span rel="pc:subject" resource="http://opendata.cz/vocabulary/cpv/2008#cpv-18911000-8">sedla</span>
   <span rel="pc:subject" resource="http://opendata.cz/vocabulary/cpv/2008#cpv-18913000-2">biče</span>
</div>

Dále můžete zveřejnit specifické údaje o veřejné zakázce dané legislativou. Tím myslíme druh řízení k veřejné zakázce (vlastnost pccz:managedAs), limit zakázky (vlastnost pccz:limit) a druh zakázky (vlastnost pccz:kind). Jejich hodnotami jsou opět zdroje reprezentující jednotlivé typy řízení, limitů a druhů zakázek.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   …
  Druh řízení: <span rel="pccz:managedAs" resource="http://purl.org/opendata-cz/public-contracts/czech#NegotiationWithPublicationManagement">Jednací s uveřejněním</span>
  Limit: <span rel="pccz:limit" resource="http://purl.org/opendata-cz/public-contracts/czech#UnderLimit">Podlimitní</span>
  Druh: <span rel="pccz:kind" resource="http://purl.org/opendata-cz/public-contracts/czech#Supplies">Dodávky</span>
</div>

Konkrétně slovník nabízí tyto možné hodnoty:

  • pccz:managedAs může mít jako hodnotu jeden z následujících zdrojů:
    • http://purl.org/opendata-cz/public-contracts/czech#OpenManagement - Otevřené řízení
    • http://purl.org/opendata-cz/public-contracts/czech#NarrowManagement - Užší řízení
    • http://purl.org/opendata-cz/public-contracts/czech#NegotiationWithPublicationManagement - Řízení jednací s uveřejněním
    • http://purl.org/opendata-cz/public-contracts/czech#NegotiationWithoutPublicationManagement - Řízení jednací bez uveřejnění
    • http://purl.org/opendata-cz/public-contracts/czech#CompetitionDialogueManagement - Soutěžní dialog
    • http://purl.org/opendata-cz/public-contracts/czech#SimpleUnderLimitManagement - Zjednodušené podlimitní řízení
  • pccz:limit může mít jako hodnotu jeden z následujících zdrojů:
    • http://purl.org/opendata-cz/public-contracts/czech#SmallAmount - Zakázka malého rozsahu
    • http://purl.org/opendata-cz/public-contracts/czech#UnderLimit - Podlimitní zakázka
    • http://purl.org/opendata-cz/public-contracts/czech#AboveLimit - Nadlimitní zakázka
  • pccz:kind může mít jako hodnotu jeden z následujících zdrojů:
    • http://purl.org/opendata-cz/public-contracts/czech#Supplies - Dodávky
    • http://purl.org/opendata-cz/public-contracts/czech#Constructions - Stavební práce
    • http://purl.org/opendata-cz/public-contracts/czech#Services - Služby

Připojte informace o zadavateli veřejné zakázky pomocí vlastnosti pc:submitter. Zadavatele uvádějte jako anonymní instanci třídy gr:BusinessEntity, tj. v samostatném elementu s RDFa atributem typeof s hodnotou gr:BusinessEntity.

<div about="http://www.mesto.cz/VZ/12345678" typeof="c:Contract">
  …
  <div rel="pc:submitter">
    <div typeof="gr:BusinessEntity">Zadavatel:
      …                 	
    </div>
  </div>

Doplňte jméno (RDF vlastnost gr:legalName) a IČ (RDF vlastnost pc:orgNumber) zadavatele.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  <div rel="pc:submitter">
    <div typeof="gr:BusinessEntity">Nabízející:
      <span property="gr:legalName">Město Ukázkové</span>
      , IČ
      <span property="pc:orgNumber">987654321</span>                 	
    </div>
  </div>

Údaje při zadání veřejné zakázky

Po zadání veřejné zakázky můžete zveřejnit další údaje. První informací je počet obdržených nabídek, k čemuž použijte RDF vlastnost pc:numberOfBids.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   …
   Počet obdržených nabídek: <span property="pc:numberOfBids" datatype="xsd:int">2</span>
</div>

Slovník umožňuje také zveřejnit jednotlivé nabídky jako instance třídy pc:Bid. Nabídku přiřadíte k veřejné zakázce pomocí vlastnosti pc:bid. Nabídek můžete přiřadit několik.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
      …
  </div>
  <div rel="pc:bid">
      …
  </div>
</div>

Samotnou nabídku umístěte do samostatného elementu (např. div) s RDFa atributem typeof s hodnotou pc:Bid.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      …
    </div>
  </div>
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      …
    </div>
  </div>
</div>

Ke každé nabídce přidejte společnost, která zaslala nabídku, pomocí vlastnosti pc:bidder. Společnost uvádějte jako anonymní instanci třídy gr:BusinessEntity, tj. v samostatném elementu s RDFa atributem typeof s hodnotou gr:BusinessEntity.

<div about="http://www.mesto.cz/VZ/12345678" typeof="c:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      <div rel="pc:bidder">
        <div typeof="gr:BusinessEntity">Nabízející:
          …                 	
        </div>
      </div>
    </div>
  </div>
  <div rel="pc:bid">
    …
  </div>
</div>

Doplňte data o nabízejícím - jeho název (RDF vlastnost gr:legalName) a IČO (RDF vlastnost pc:orgNumber).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      <div rel="pc:bidder">
        <div typeof="gr:BusinessEntity">Nabízející:
          <span property="gr:legalName">První jezdecká a.s.</span>
          , IČ
          <span property="pc:orgNumber">789654123</span>                 	
        </div>
      </div>
    </div>
  </div>
</div>

Kromě nabízejícího uvádějte k nabídce také nabídnutou cenu pomocí vlastnosti pc:bidPrice. Cenu uvádějte jako anonymní instanci třídy gr:PriceSpecification, tj. v samostatném elementu s RDFa atributem typeof s hodnotou gr:PriceSpecification.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      <div rel="pc:bidder">
        <div typeof="gr:BusinessEntity">Nabízející:
          <span property="gr:legalName">První jezdecká a.s.</span>
          , IČ
          <span property="pc:orgNumber">789654123</span>                 	
        </div>
      </div>
      <div rel="pc:bidPrice">
        <div typeof="gr:PriceSpecification">Nabídnutá cena:
          …
        </div>
      </div>
    </div>
  </div>
</div>

K ceně doplňte hodnotu (RDF vlastnost gr:hasCurrencyValue), měnu (RDF vlastnost gr:hasCurrency) a zda je cena s DPH (RDF vlastnost gr:valueAddedTaxIncluded).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      <div rel="pc:bidder">
        <div typeof="gr:BusinessEntity">Nabízející:
          <span property="gr:legalName">První jezdecká a.s.</span>
          , IČ
          <span property="pc:orgNumber">789654123</span>                 	
        </div>
      </div>
      <div rel="pc:bidPrice">
        <div typeof="gr:PriceSpecification">Nabídnutá cena:
          <span property="gr:hasCurrencyValue" datatype="xsd:int">1.500.000</span>
          <span property="gr:hasCurrency" content="CZK">Kč</span>
          <span property="gr:valueAddedTaxIncluded" content="false" datatype="xsd:boolean">bez daně</span>
        </div>
      </div>
    </div>
  </div>
</div>

Vítěznou nabídku nepřiřazujte k veřejné zakázce pomocí RDF vlastnosti pc:bid ale pomocí vlastnosti pc:acceptedBid. Samotnou vítěznou nabídku popište stejně jako ostatní nabídky, tj. jako instance třídy pc:Bid.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Vítězná nabídka:
  <div rel="pc:acceptedBid">
    <div typeof="pc:Bid">
      …
    </div>
  </div>
</div>

K veřejné zakázce můžete uvést dokumenty, které se ji týkají (zadávací dokumentace, protokol z rozbalování obálek s nabídkami, smlouva).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   …
   <div><a rel="pc:document" href="http://www.mesto.cz/docs/12345678_popis.doc">Popis zakázky (DOC)</a></div>
   <div><a rel="pc:document" href="http://www.mesto.cz/docs/12345678_smlouva.pdf">Smlouva (PDF)</a></div>
</div>

Cenu za zakázku dohodnutou s vítězným dodavatelem na smlouvě můžete uvést pomocí vlastnosti pc:contractPrice (opět jako instanci třídy gr:PriceSpecification).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  <div rel="pc:contractPrice">
    <div typeof="gr:PriceSpecification">Smluvní cena:
      <span property="gr:hasCurrencyValue" datatype="xsd:int">1.000.000</span>
      <span property="gr:hasCurrency" content="CZK">Kč</span>
      (
      <span property="gr:valueAddedTaxIncluded" content="false" datatype="xsd:boolean">bez daně</span>
      )
    </div>
  </div>
</div>

Údaje při dokončení realizace veřejné zakázky

Dodatkovou cenu (cena navíc vzniklá v průběhu realizace zakázky) a dotaci (částka z celkové konečné ceny zakázky, která byla pokryta nějakou dotací) můžete uvést pomocí vlastnosti c:supplementPrice, resp. pc:subsidy (opět jako instance třídy gr:PriceSpecification).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  <div rel="pc:supplementPrice">
    <div typeof="gr:PriceSpecification">Dodatková cena:
      <span property="gr:hasCurrencyValue" datatype="xsd:int">150.000</span>
      <span property="gr:hasCurrency" content="CZK">Kč</span>
    </div>
  </div>
  <div rel="pc:subsidy">
    <div typeof="gr:PriceSpecification">Dotace:
      <span property="gr:hasCurrencyValue" datatype="xsd:int">20.000</span>
      <span property="gr:hasCurrency" content="EUR">€</span>
    </div>
  </div>
</div>

Také můžete pomocí vlastností pc:quality a pc:onSchedule uvést, zda byla splněna očekávaná kvalita při realizaci veřejné zakázky a zda byly dodrženy termíny.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Splněna v termínu: <span property="pc:onSchedule" content="false" datatype="xsd:boolean">ano</span>
  Splněna požadovaná kvalita: <span property="pc:quality" content="true" datatype="xsd:boolean">ne</span>
</div>
